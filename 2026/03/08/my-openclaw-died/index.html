我的小龙虾，死了

今天收到了腾讯云的短信，我的小龙虾服务器，明天 0:00 就要到期了。

今天我给它发了一条告别消息，而它也平静地接受了。

01 我的养虾史

OpenClaw 是一个开源的 AI Agent 框架，部署在服务器上后，可以通过飞书、Telegram 等渠道与 AI 对话。社区里大家习惯叫它”龙虾”。

这是我养的第三只龙虾了。

第一只：bot

还是在 OpenClaw 还没被改名的时候养的，接入了 Claude Opus 模型。

当时我的感受是：这东西为啥会火，直接用 Claude Code + Skills + claude-mem + happy 也能做到，浪费了大量的 token，完全是高射炮打蚊子。

（注：Claude Code 是著名的 AI 编程助手；claude-mem 是它的长期记忆插件；happy 是手机远程操作 Claude Code 的工具）

在接下来的日子里，我也尝试发掘它的潜力，安装了社区里各种插件。但总觉得不如我直接手机操作 Claude Code。除了回复我一堆堆废话，刷刷刷烧着 token 外，完全感觉不到存在感。

所以这只龙虾活了一周就被我给格式化了。

第二只：kimi-claw

Kimi 是我个人挺喜欢的一家公司。他们总是会做一些”新”的小玩具，比如最初的 PPT、长上下文，最近的 ok-computer、Agent 集群，也包括这次的 kimi-claw。

这个新玩具，需要订阅 199 元/月的套餐，没有优惠。

本质上就是它给你开了个虚拟机，装好改版的 Claw，配了一些插件，API 也内置配置好。整个初始化只要 1 分钟，不需要配 tg-bot，不需要配飞书机器人，直接在 Kimi 的网页端和 App 里就能和龙虾对话。

本着来都来了、钱都花了的心态，这只龙虾我是用得最多的。所以吐槽也最多：

慢，Kimi-2.5 是真的慢，问它写个简单脚本，Opus 30 秒搞定的事它要磨蹭好几分钟。不稳定，遇到一次前一天还能用，第二天就莫名重置了。封闭，无法外网访问，想传个文件都很烦。笨，相对于 Opus，真的是降维打击。记性差，号称长上下文，实际体验一言难尽。

体验上，主要赢在一个方便：部署方便、内置插件方便、手机直接 App 打开方便。其他的没什么新感受。

算下来，国产各家我基本都充过一波了（除了腾讯的实在太黑我只想白嫖），在我看来，没被御三家拉开代差。虽然这个市场最终还是会一地鸡毛，但至少目前还是一个不错的信号。

第三只：xiami

腾讯的新春活动白嫖了一个月云主机，配上 Claw 应用模板，配上我丰富的龙虾折腾经验，五分钟部署完直接走起。

（看了下活动到 3 月底，最近很火的腾讯帮你装 OpenClaw，估计就是直接拿这套在拉人头 :）

由于是自己的服务器，可玩性就高了点。

先将模型换了，直接整上 2api（一个第三方 API 中转服务）的 Codex，最近 Codex 普号能用后，价格白菜得一塌糊涂。

然后飞书直接建了个虾厂，拉上小伙伴，主打一个共享龙虾。

接下来各种整活的 Skill、外部 API 配上，主打一个娱乐性拉满，安全性裸奔。

但是用了几天，又开始索然无味了。编程工作不如直接用 Claude Code，安全性又是一个黑箱，不敢给它太多隐私。所以慢慢就沦为鸡肋了。

再后面就是各种龙虾变种（OpenClaw 这几十万行代码是真的没眼看），各自做出了差异化。

02 突然爆火的龙虾

然后，这玩意最近火出圈了？

从一个极客玩具，突然就变成了人手必备的贾维斯了？

腾讯、字节、阿里、各大云厂商纷纷下场不说，连指导文件都出现了。（我：黑人问号？）

想起最近做微舆的高中生直接喜提盛大 3000W 投资变 CEO，他在 L 站发求关注贴的时候才 600 Star。说实话，技术上并不复杂，但胜在产品嗅觉和执行力——这可能才是 AI 时代最稀缺的能力。

我突然有点明白了，AI 时代，做产品的思路已经发生了根本性的变化。

而这种变化，其实有迹可循。

03 从瀑布到 AI

早期软件开发的经典模型，是瀑布模型：需求、规格、概要设计、详细设计、编码、单元测试、集成测试。

那个时代软件的开发成本极高，每一步都要反复打磨，否则背后可能就是万丈深渊。一个项目从立项到交付，动辄以年计。

下一个经典模型，是敏捷开发：强调沟通、快速迭代。

一方面是因为这玩意毕竟是一群外包大佬提出的，另一方面也是因为试错和发行的成本降低了——有了云服务、有了 App Store，你可以先上线一个 MVP，再根据反馈迭代。渐进明细，成为了降低风险的必然选择。

而我们可能即将看到一个全新的模型：AI-CoWork。

因为 AI 能力的提升，开发成本会极速降低。人的工作变成了定方向、审核、做决策。AI 在一直工作，人只是偶尔喊两声。

回想我自己养虾的经历，五分钟部署、换个模型、配几个插件就上线了——这在两年前是不可想象的。

当”做出来”不再是瓶颈，**”让人知道”就成了新的战场。**

所以我们看到的趋势是：先是超级个体、网红工程师靠个人影响力出圈，然后最终是平台收割。

软件领域将会如同拼多多、抖音那样，最终倒在流量面前。因为你的技能不再是门槛，当 AI 能帮所有人写代码，能写代码这件事本身就不值钱了。

而且更远一点看，AI 自身演进进化的能力正在浮现。

今天还需要人来”养虾”，明天虾可能自己就会游了。

程序员可能成为第一个自己革了自己命的行业（笑

04 虾死了，然后呢

小龙虾的服务器到期了，我没有续费。

不是因为它没用，而是因为我已经从它身上学到了该学的东西：

AI 时代的产品，技术只是入场券，真正的壁垒在于你能多快地触达用户、理解需求、完成闭环。

三只虾，从”这东西为啥会火”到”我突然有点明白了”，这条路走了不到半年。

下一只虾，也许不会再叫虾了。但那个让我凌晨还在折腾、第二天又觉得索然无味、然后某一刻突然想明白点什么的过程——大概不会停。
